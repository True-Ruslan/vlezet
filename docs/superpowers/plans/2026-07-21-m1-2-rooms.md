# M1.2 — Rooms and Usable Area Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Derive enclosed rooms from explicit wall topology, calculate trustworthy usable interior polygons and areas from physical wall thickness, and let users name rooms without persisting duplicate geometry.

**Architecture:** `@vlezet/geometry` converts valid atomic wall edges into a deterministic planar half-edge graph, extracts bounded faces, offsets each face inward by half of each boundary wall's thickness, validates the resulting polygon, and derives room identity/label points. `RoomAnnotation` remains the only persistent room data; geometry and area are recomputed from walls on every relevant document change.

**Tech Stack:** TypeScript 6.0.3, Vitest 4, Zustand 5, React 19, Konva/react-konva, Next.js 16.

## Global Constraints

- Rooms are derived, never persisted as polygons.
- Authoritative area uses inner usable wall boundaries, not centre-lines.
- Canonical units are millimetres and square millimetres; UI displays square metres.
- Invalid topology must not produce authoritative room area.
- Room identity must be deterministic for identical topology and stable through ordinary vertex movement that preserves the same boundary vertex identities.
- Concave simple rooms must be supported; collapsed/self-intersecting usable offsets are rejected.
- M1.2 does not add doors/windows, furniture, persistence backend, 3D, or AI.
