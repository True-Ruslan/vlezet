# M5 Spatial 3D Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a deterministic, read-only 3D projection of the existing `VlezetDocument`, beginning with walls, opening placeholders and room floors, then expose it through an isolated Three.js viewer with safe 2D/3D switching.

**Architecture:** A new framework-independent `@vlezet/spatial` package converts the authoritative document plus existing geometry derivations into a renderer-neutral `SpatialScene`. `apps/web` consumes only that neutral scene; Three.js owns rendering/camera/resources but never derives apartment truth or mutates the document.

**Tech Stack:** TypeScript 6, pnpm/Turborepo, Vitest, Next.js 16/React 19, `@vlezet/domain`, `@vlezet/geometry`, plain Three.js + OrbitControls.

## Global Constraints

- Millimetres remain the canonical spatial unit; `document x → scene X`, `document y → scene Z`, vertical height → scene Y.
- `VlezetDocument` remains the only persisted geometry source of truth; M5.1 adds no schema migration.
- No direct geometry editing in 3D.
- Opening width/offset are authoritative; vertical door/window heights are schematic until explicit document metadata exists.
- 2D/3D switching is ephemeral UI state and must not create history entries or autosave document mutations.
- WebGL/3D failures must never block the 2D editor or project startup.
- Every code-bearing head must pass frozen install, unit tests, typecheck, lint and production build before browser acceptance.

- [ ] **Step 3: Implement minimal neutral types and projection**

Core API shape:

```ts
export const DEFAULT_WALL_HEIGHT_MM = 2700;

export type Point3 = Readonly<{ x: number; y: number; z: number }>;
export type SpatialWallSegment = Readonly<{
  id: string;
  wallId: string;
  startOffsetMm: number;
  endOffsetMm: number;
  center: Point3;
  lengthMm: number;
  thicknessMm: number;
  heightMm: number;
  rotationYRad: number;
}>;
export type SpatialOpeningMarker = Readonly<{
  id: string;
  openingId: string;
  wallId: string;
  kind: "door" | "window";
  center: Point3;
  widthMm: number;
  wallHeightMm: number;
  rotationYRad: number;
}>;
export type SpatialFloor = Readonly<{
  id: string;
  roomId: string;
  polygon: readonly Point3[];
}>;
export type SpatialScene = Readonly<{
  wallSegments: readonly SpatialWallSegment[];
  openingMarkers: readonly SpatialOpeningMarker[];
  floors: readonly SpatialFloor[];
}>;
```

Projection must use existing geometry helpers rather than reimplementing opening/room mathematics.
