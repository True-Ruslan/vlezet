# M1.3 — Doors and Windows Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add doors and windows as dimensionally correct openings hosted by stable semantic wall runs, with true visual gaps, door swing planning, exact inspector editing, validation, selection, undo/redo, and stable placement through T-junction subdivision.

**Architecture:** Persistent `Opening` data already lives in schema v2 as `wallId + offset + width + kind`. `@vlezet/editor-core` owns opening validation and mutations. `@vlezet/geometry` derives visible wall intervals and opening world geometry. The Canvas renders physical wall segments around gaps and overlays door/window symbols; UI never stores pixel placement.

## Product defaults

- Door default width: 900 mm.
- Window default width: 1200 mm.
- Placement preview is centred on pointer projection and clamped to wall bounds.
- Door default: hinge at opening start, swing to wall-left side.
- Exact width/offset/orientation are editable in inspector.
- Openings cannot overlap or straddle internal T-junctions in M1.
