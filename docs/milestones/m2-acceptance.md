# M2 — Furnishing and Fit Browser Acceptance

Проверка выполняется на ветке `feat/m2-furnishing-fit` после успешного CI.

## Запуск

```bash
git fetch
git checkout feat/m2-furnishing-fit
git pull
pnpm install --frozen-lockfile
pnpm dev
```

Открыть адрес, который напечатает Next.js.

## 1. Подготовить комнату

1. Нарисовать замкнутую комнату примерно `6000 × 4000 мм`.
2. Убедиться, что появилась заливка комнаты и полезная площадь.
3. Добавить дверь шириной `900 мм`.
4. Оставить дверь открывающейся внутрь комнаты.

Ожидается:

- комната определяется автоматически;
- дверь создаёт настоящий разрыв в стене;
- зона открывания двери видна на плане.

## 2. Разместить спальню

Из левого каталога последовательно добавить:

- `Двуспальная кровать`;
- `Шкаф`;
- `Рабочий стол`.

Ожидается:

- после выбора карточки появляется полупрозрачный preview;
- клик на плане создаёт предмет с реальными размерами;
- новый предмет сразу выбран;
- справа открывается inspector предмета;
- в toolbar растёт счётчик предметов.

## 3. Проверить статус «Не влезает»

1. Перетащить шкаф поверх кровати.
2. Затем поставить шкаф в сектор открывания двери.

Ожидается:

- предмет получает красный контур;
- inspector показывает `Не влезает`;
- причина отдельно сообщает о пересечении с предметом или блокировке двери;
- состояние не запрещает редактирование: предмет можно продолжить двигать.

## 4. Проверить статус «Влезает вплотную»

1. Убрать физическое пересечение шкафа с другими предметами.
2. Поставить шкаф так, чтобы его корпус помещался, но рекомендуемые `800 мм` перед фасадом упирались в стену или другой предмет.

Ожидается:

- контур становится янтарным;
- inspector показывает `Влезает вплотную`;
- причина сформулирована как рекомендация по зоне использования, а не как нормативное нарушение.

## 5. Проверить статус «Влезает»

Переместить шкаф туда, где корпус и рекомендуемая зона использования свободны.

Ожидается:

- inspector показывает `Влезает`;
- список конфликтов исчезает;
- результат обновляется сразу после завершения перемещения.

## 6. Drag, snapping и guides

1. Перетаскивать стол рядом со шкафом и кроватью.
2. Проверить выравнивание по краям предметов.
3. Проверить выравнивание по центрам.
4. Отвести предмет от других объектов и проверить grid snapping.

Ожидается:

- движение плавное;
- появляются направляющие выравнивания;
- приоритет: край → центр → сетка;
- один завершённый drag создаёт ровно одну операцию undo.

## 7. Поворот и изменение размеров

1. Повернуть стол через handle Transformer.
2. Проверить привязку угла к шагам `15°` и основным углам.
3. Нажать `R` и проверить поворот выбранного предмета на `90°`.
4. Изменить размер предмета corner handles.
5. В inspector ввести точные ширину, глубину, X, Y и угол.

Ожидается:

- центр предмета не «прыгает» при повороте;
- после resize реальные размеры отображаются в миллиметрах;
- invalid numeric input не меняет документ;
- undo полностью восстанавливает предыдущую геометрию.

## 8. Измерения

Выбрать любой предмет внутри комнаты.

Ожидается:

- на Canvas видны ширина и глубина;
- показываются расстояния спереди, справа, сзади и слева;
- inspector показывает те же расстояния в миллиметрах;
- ближайшим препятствием может быть стена, другой предмет или сектор двери.

## 9. Рекомендуемые зоны

Выбрать кровать, шкаф и стол по очереди.

Ожидается:

- выбранный предмет показывает пунктирную functional-clearance zone;
- зоны асимметричны и ориентируются вместе с предметом;
- изменение зазоров в inspector немедленно влияет на fit status после применения.

## 10. Дублирование и удаление

1. Выбрать предмет и нажать `Ctrl/Cmd + D`.
2. Проверить, что копия появляется со смещением `200 × 200 мм`.
3. Нажать `Delete` или `Backspace`.

Ожидается:

- копия получает новый ID и становится выбранной;
- исходный предмет не меняется;
- удаляется только выбранная копия;
- undo/redo корректно возвращает и снова удаляет её.

## 11. Свой предмет

1. Выбрать `Свой предмет`.
2. Разместить его на плане.
3. В inspector переименовать в `Пианино`.
4. Установить размер `1450 × 600 мм` и точный угол.

Ожидается:

- custom object проходит через тот же fit engine;
- размеры, коллизии, snapping, измерения и undo работают так же, как для каталожного предмета.

## 12. История

Последовательно выполнить:

1. placement;
2. drag;
3. rotate;
4. resize/update;
5. duplicate;
6. delete.

Затем отменить все действия и повторить их через redo.

Ожидается:

- каждое смысловое действие занимает одну позицию истории;
- промежуточные pointer events не засоряют undo;
- документ после полного redo совпадает с состоянием до undo.

## 13. Ошибочная планировка

Создать неявное X-пересечение стен или другую topology error.

Ожидается:

- Vlezet показывает ошибку планировки;
- furniture fit не заявляет достоверное `Влезает`;
- предметы остаются доступными для редактирования.

## Итоговый критерий

M2 считается принятым вручную, если пользователь может обставить комнату, получить объяснимые статусы fit, точно отредактировать предметы и предсказуемо отменить каждое действие без повреждения стен, комнат и проёмов.
